# Прототип музыкального сервиса с рекомендательной системой, использующей психологические черты пользователей

Это прототип веб-приложения музыкального сервиса, который решает проблему холодного старта (когда у нового пользователя нет истории прослушиваний) при помощи данных о чертах личности пользователя из модели «Большой пятерки» (экстраверсия и открытость новому опыту).

## Требования к системе

Перед началом убедитесь, что ваша среда соответствует следующим требованиям:

- **Операционная система**: Windows 10/11
- **Python**: версия 3.9 или выше
- **PostgreSQL**: версия 17 или выше
- **Браузер**: Google Chrome (последняя стабильная версия)
- **Дополнительно**: ChromeDriver, совместимый с вашей версией Chrome
- **Оперативная память**: не менее 4 ГБ (рекомендуется 8 ГБ)
- **Дисковое пространство**: не менее 2 ГБ 

## Установка и развёртывание

### Шаг 1. Клонирование репозитория

```bash
git clone https://github.com/IreneRadix/Big_five_music_recommender_prototype.git
```

### Шаг 2. Установка и настройка PostgreSQL

Установите и настройте PostgreSQL (Официальный сайт: https://www.postgresql.org/download/windows/). Создайте базу данных music_db при помощи скрипта music_db_v3.

### Шаг 3. Настройка переменных окружения

Создайте файл .env в корне проекта и напишите туда Ваше имя пользователя и пароль PostgreSQL (DB_USER и DB_PASSWORD соответственно). Также добавьте в файл ваш ключ vk_api под именем переменной VK_API_KEY.

### Шаг 4. Установка зависимостей

Запустите в терминале
```
pip install -r requirements.txt
```

### Шаг 5. Запуск парсера

Запустите файл vk_parser_start, и ждите, когда база данных заполнится.

### Шаг 6. Запуск веб-сервера

Запустите в терминале команду flask run.
